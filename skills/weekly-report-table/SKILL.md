# 驱动小组周报总结表格生成规范

当用户提供驱动小组成员的周报并要求整理为小组总结表格时，必须遵循以下所有规则。

---

## 1. 总体原则

- 从**小组整体角度**进行总结，而非个人视角。
- **不包含任何成员姓名**，只保留任务内容本身。
- 将不同成员对同一项目的工作**合并到同一条目**下。

## 2. 输入来源

- 输入为驱动小组三位成员（朱兴瑞、王霆、朱日芃）的个人周报。
- 输入格式通常为按负责人分列的"本周任务"和"下周任务"。
- 输入来源可以是：
  - Markdown 文件
  - 截图图片
  - 飞书文档链接
- 当输入为飞书文档链接时，使用飞书 MCP 工具 `docx_v1_document_rawContent` 获取内容，`document_id` 从 URL 路径末段提取。

## 3. 输出要求

每次生成**两份输出**：

### 3.1 本地 Markdown 文件

- 文件命名格式：`驱动小组周报表格_MMDD.md`（`MMDD` 为周报对应日期）。
- 使用 Markdown 表格语法。

### 3.2 飞书云文档

- 使用飞书 MCP 工具 `docx_builtin_import` 创建。
- 文件名格式：`驱动小组周报表格_MMDD`。
- **必须使用 HTML `<table>` 语法**（飞书不支持 Markdown 表格内的 `<br>` 换行）。
- 生成后返回飞书文档链接给用户。

## 4. 表格结构

表格固定为三列、四行分类：

| 任务分类 | 本周任务汇总 | 下周任务安排 |
|----------|--------------|--------------|
| 重点项目开发 | ... | ... |
| 技术选项与优化 | ... | ... |
| 问题定位与支持 | ... | ... |
| 调试测试文档 | ... | ... |

## 5. 任务分类定义

### 5.1 重点项目开发

核心产品项目的开发与推进工作，包括但不限于：

- 系统移植与适配（如 RK3588 RIGOLOS）
- 产品项目开发（如草原雕 II、金雕、玄铁重剑等）
- 驱动移植（PCIe、SDIO、LCD、Touch 等）
- MCU 固件开发与升级

### 5.2 技术选项与优化

技术方案调研、代码优化和模板改进，包括但不限于：

- 技术方案调研与文档（如扩屏方案、USBTMC 方案）
- MCU 模板优化（如升级功能、日志功能、FPGA 下载等）
- HAL 库优化与适配
- 代码框架整合与重构

### 5.3 问题定位与支持

Bug 定位、客户问题处理和跨项目技术支持，包括但不限于：

- 客户问题处理（如华为 DS7W 等）
- 硬件问题定位（如通信异常、枚举失败等）
- 跨项目协助与技术支持
- 压测问题联调

### 5.4 调试测试文档

测试报告、技术文档等输出物，包括但不限于：

- 测试文档（如网卡测试文档）
- 技术方案文档（如调研报告、方案草稿）
- 教程与操作指南
- HAL 库更新文档

## 6. 内容格式规范

### 6.1 层级结构

每个分类下按**项目名称**分组，项目下列出具体任务条目：

```
• **项目名称**：
    - 具体任务 1
    - 具体任务 2
```

### 6.2 无内容处理

若某分类下本周或下周无对应任务，填写 `-`。

### 6.3 本地 Markdown 文件格式

- 使用 `<br>` 实现表格单元格内换行。
- 使用 `<br><br>` 分隔不同项目。
- 使用 `&nbsp;&nbsp;` 实现子条目缩进。
- 项目名称使用 `**加粗**` 显示。

示例：

```markdown
| **重点项目开发** | • **RK3588 RIGOLOS 适配**：<br>&nbsp;&nbsp;- PCIe 控制器驱动移植完成<br>&nbsp;&nbsp;- XDMA 设备驱动移植完成<br><br>• **草原雕 II**：<br>&nbsp;&nbsp;- sufd 库 deinit 失败问题修复 | • **RK3588 RIGOLOS 适配**：<br>&nbsp;&nbsp;- SDIO 控制器驱动移植适配 |
```

### 6.4 飞书文档格式

**必须使用 HTML `<table>` + `<ul>/<li>` 语法**，这是飞书唯一能正确渲染表格内多行列表的方式。

- 表头使用 `<th>`，并加上 `align="center"` 实现居中对齐，内容用 `<b>` 加粗。
- 每个分类行使用 `<tr>`，第一列（任务分类）使用 `<td align="center">` 实现居中对齐，第二三列为任务内容。
- 单元格内使用 `<ul>/<li>` 嵌套列表实现项目和子条目的层级关系。
- 项目名称使用 `<b>` 加粗。

**飞书导入 API 的已知限制**：

- `bgcolor` 属性**不生效**，无法通过导入设置表头背景色。
- `style` 属性**不生效**，CSS 样式不被支持。
- `align="center"` **可以生效**，支持文字居中。
- 因此表头背景色需要在飞书文档中**手动设置**（导入后选中表头行 → 设置单元格背景色为淡紫色 `#E8D5F5`）。

示例：

```html
<table>
<tr>
<th align="center"><b>任务分类</b></th>
<th align="center"><b>本周任务汇总</b></th>
<th align="center"><b>下周任务安排</b></th>
</tr>
<tr>
<td align="center">重点项目开发</td>
<td>
<ul>
<li><b>RK3588 RIGOLOS 适配</b>：
<ul>
<li>PCIe 控制器驱动移植完成</li>
<li>XDMA 设备驱动移植完成</li>
</ul>
</li>
<li><b>草原雕 II</b>：
<ul>
<li>sufd 库 deinit 失败问题修复</li>
</ul>
</li>
</ul>
</td>
<td>
<ul>
<li><b>RK3588 RIGOLOS 适配</b>：
<ul>
<li>SDIO 控制器驱动移植适配</li>
</ul>
</li>
</ul>
</td>
</tr>
</table>
```
