# 反面案例与纠正

来源于常见抽检不通过情形（含 AMP 跨域通信测试类任务）。

## 不通过模式

| 问题 | 典型错误写法 | 为何被拒 |
|------|--------------|----------|
| 场景过简 | 「搭建 XX 测试环境」（16 字） | 无业务背景、无目的、无 AI 价值 |
| 动作过简 | 与场景同句重复（30 字） | 无步骤、无工具、无法映射产出 |
| 内容偏差 | 只交付「环境+方案文档」 | 计划要求「验证过程」，缺执行与结果 |
| 证明空洞 | 链接有效但文档只有提纲 | 审核点开看不到数据/截图/结论 |
| ROI 夸大 | 小任务选 8h+ 无依据 | 触发人工复核质疑 |

## 纠正对照

| 错误 | 改为 |
|------|------|
| 搭建 AMP 测试环境 | RIGOLOS 在 RK3588 上需完成 AMP 通信**验证过程**（含实机执行与数据），因…使用 Cursor… |
| 设计热点定位方案 | 步骤1 SSH 采集拓扑…步骤3 执行 6 组用例 N=10000…步骤4 AI 整理对比表… |
| 输出测试文档 1 份 | 飞书文档 2 份、对比表 15+ 张、画板 10+ 张；PERIOD \|p99\|=3µs；RPMG max=4648µs… |
| 仅概要页链接 | 主报告链接（§2 含六用例数据表与截图）优先填写 |

## 计划标准 ↔ 成果类型速查

| 计划标准关键词 | 输出成果必须包含 |
|----------------|------------------|
| 验证过程 | 用例执行记录 + 实测指标 + 通过/失败/根因 |
| 分析报告 | 结论段 + 支撑数据表/图 |
| 方案设计 | 方案文档 + （若标准仅设计则注明范围；若还含验证则加执行） |
| 环境搭建 | 环境说明 + （若标准仅搭建则写清边界；通常还需后续验证） |
| 代码交付 | 仓库/文件 + 功能说明 + 测试或运行证据 |

## Agent 处理「上次被驳回」

1. 读取用户提供的驳回说明（如 fail.txt）
2. 逐条映射到 checklist 未勾选项
3. 先补飞书证明（缺数据/截图），再重写四段
4. 显式写一句对齐计划标准：「本次交付覆盖…验证执行…结果分析…」
