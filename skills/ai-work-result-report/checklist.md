# 提交前自检清单

全部勾选后再让用户提交。任一项未满足须修改文案或补充飞书证明。

## 计划对齐

- [ ] 已写明**计划完成标准**或等价目标（非自拟模糊目标）
- [ ] 「输出成果」含**执行/验证/分析结果**，不仅是环境、方案、计划
- [ ] 若标准为「验证过程」，证明链接中有**实测数据或明确验证结论**

## 四段质量

- [ ] **使用场景** ≥ 约 80 字，含背景、问题、用 AI 原因；不与「如何使用」逐句重复
- [ ] **如何使用** 含 ≥ 3 个可区分步骤；每步有**工具名**；每步有**对应产出**（形态+数量）
- [ ] **输出成果** 含**数量词**（份/张/个/组），无单独「已完成」「已优化」
- [ ] 至少 1～3 条**可核对**的数据、结论或判定

## 证明链接

- [ ] 链接为飞书/组织内可访问地址（非仅本地路径）
- [ ] 用无权限或同事账号点开**无需申请**即可阅读
- [ ] 链接内容与第 3 节描述一致（表、截图、章节存在）
- [ ] 本地 IDE/终端截图已汇总进飞书（若曾只有本地材料）

## 动作 ↔ 证据映射

- [ ] 每个「步骤N」能在证明文档中找到对应章节、图或表
- [ ] 采取的动作 ≠ 使用场景的复制粘贴

## ROI

- [ ] 节省时间档位与文档规模、轮次、数据量**相称**
- [ ] 理由句不与交付清单明显矛盾（如只交付 1 页却写 8h+ 需有说明）

## 措辞避雷

- [ ] 无仅 16～30 字的场景/动作敷衍句
- [ ] 无「搭建环境」「设计方案」作为**唯一**成果（当标准是完整验证/交付时）
